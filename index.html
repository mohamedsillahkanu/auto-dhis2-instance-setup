⚡ Method 2: Build & Deploy via Docker Registry

FOR DEVELOPERS

Build your custom Docker image and push it to Docker Hub, then deploy to Render.

📚 What is Docker Registry?

A Docker Registry is like GitHub for Docker images. You push your built image there, and Render pulls it to deploy. Popular registries:

  • Docker Hub (hub.docker.com) - Free, 1 private repo
  • GitHub Container Registry (ghcr.io) - Free unlimited
  • Render Registry - Built-in with Render

📋 Step-by-Step Instructions

Step 1: Create Docker Hub Account

Go to hub.docker.com and create a free account

Step 2: Download Your Files
Step 3: Build & Push Image

Place both files in a folder and run these commands:

# Login to Docker Hub
docker login

# Build your image (replace 'yourusername')
docker build -t yourusername/dhis2-custom:latest .

# Push to Docker Hub
docker push yourusername/dhis2-custom:latest

✅ Your personalized commands:

docker build -t /dhis2-custom:latest .
docker push /dhis2-custom:latest
Step 4: Deploy to Render

In Render, choose "Deploy from Docker Registry" and use:

Docker Image:
yourusername/dhis2-custom:latest
Step 5: Done! 🎉

Render will pull your custom image and deploy DHIS2 with your configuration baked in!

⚠️ Requirements

  • Docker Desktop installed on your computer
  • Basic command line knowledge
  • Free Docker Hub account

✅ Benefits of This Method

  • Your dhis.conf is baked into the image - no manual configuration needed
  • Faster deployment - Render just pulls the ready image
  • Can deploy the same image to multiple platforms
  • No GitHub required
